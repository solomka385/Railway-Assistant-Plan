# Матрица взаимодействия подразделений железнодорожного транспорта

## Обзор документа

Данный документ содержит подробную матрицу взаимодействия между подразделениями железнодорожного транспорта, описывает протоколы коммуникации, каналы связи и процедуры координации при различных ситуациях.

---

## 3. Процедуры координации

### 3.1 Процедура объявления аварийной ситуации

1. **Обнаружение аварии**
   - Система мониторинга или очевидец фиксирует аварию
   - Диспетчер получает сигнал

2. **Первичная оценка**
   - Диспетчер оценивает тип и масштаб аварии
   - Определяет уровень угрозы

3. **Объявление аварии**
   - Диспетчер объявляет аварию по каналу критической важности
   - Передаёт сигнал в ЦЧС и ДАВС

4. **Активация плана**
   - ДАВС активирует соответствующий план ликвидации
   - ЦЧС начинает мониторинг ситуации

5. **Вызов сил**
   - ДАВС вызывает необходимые подразделения
   - ДЦУП останавливает движение на участке

### 3.2 Процедура координации при крупной аварии

1. **Создание оперативного штаба**
   - ДАВС создаёт оперативный штаб
   - МКБКС разворачивается на месте аварии

2. **Назначение ответственных**
   - Назначается руководитель операции
   - Определяются ответственные за направления

3. **Планирование работ**
   - Разрабатывается план ликвидации
   - Определяются приоритеты и сроки

4. **Координация действий**
   - Оперативный штаб координирует все подразделения
   - Регулярные брифинги и отчёты

5. **Контроль выполнения**
   - Постоянный мониторинг хода работ
   - Корректировка плана при необходимости

### 3.3 Процедура передачи информации

1. **Сбор информации**
   - Все подразделения собирают информацию о своей зоне ответственности
   - Данные передаются в оперативный штаб

2. **Обработка информации**
   - Оперативный штаб обрабатывает и анализирует данные
   - ЦЧС моделирует развитие ситуации

3. **Распространение информации**
   - Обработанная информация распространяется между подразделениями
   - Руководство получает регулярные отчёты

4. **Информирование общественности**
   - ДЦС информирует пассажиров
   - Пресс-служба готовит заявления для СМИ

## 4. Специфические сценарии взаимодействия

### 4.1 Взаимодействие при сходе подвижного состава

**Основные участники:**
- ДАВС — координация
- ДЦУП — управление движением
- ЦЧС — мониторинг
- ДИ — восстановление пути
- ДРП — замена рельсов
- МЧ — расчистка
- МАБ — медицинская помощь

**Последовательность взаимодействия:**
1. ДЦУП → ДАВС: сигнал о сходе
2. ДАВС → ЦЧС: запрос мониторинга
3. ЦЧС → ДАВС: данные о ситуации
4. ДАВС → ДИ, ДРП, МЧ: вызов бригад
5. ДАВС → МАБ: вызов медицинской помощи
6. ДИ → ДАВС: отчёт о прибытии
7. ДРП → ДАВС: отчёт о работах
8. ДАВС → ДЦУП: прогноз восстановления

### 4.2 Взаимодействие при пожаре на вокзале

**Основные участники:**
- ДЦС — координация эвакуации
- ДГПС — принятие решений
- МЧС — пожаротушение
- МАБ — медицинская помощь
- ФГП — охрана порядка
- ДЦУП — управление движением

**Последовательность взаимодействия:**
1. ДЦС → ДГПС: сигнал о пожаре
2. ДГПС → ДЦС: решение об эвакуации
3. ДЦС → МЧС: вызов пожарных

5. ДЦС → ФГП: запрос охраны
6. ДЦС → ДЦУП: информация о ситуации
7. МЧС → ДЦС: отчёт о тушении
8. ДЦС → ДЦУП: прогноз восстановления

### 4.3 Взаимодействие при обрыве контактной сети

**Основные участники:**

7. ДИ → ЦУЭД: отчёт о восстановлении

### 4.4 Взаимодействие при террористическом акте

## 5. Регламент взаимодействия при различных уровнях угрозы

### 5.1 Уровень угрозы: Критический

**Характеристики:**
- Угроза жизни людей
- Полная блокировка движения
- Разрушение инфраструктуры

**Требуемые действия:**
- Немедленное объявление ЧС
- Активация всех планов ликвидации
- Создание оперативного штаба
- Привлечение всех доступных ресурсов
- Непрерывная координация

**Ключевые подразделения:**
- ДАВС, ЦЧС, ДЦУП, РСЧС, МЧС, УМЖД

### 5.2 Уровень угрозы: Высокий

**Характеристики:**
- Угроза здоровью людей
- Частичная блокировка движения
- Значительные повреждения инфраструктуры

**Требуемые действия:**
- Объявление аварии
- Активация соответствующего плана
- Координация основных подразделений
- Привлечение необходимых ресурсов

**Ключевые подразделения:**
- ДАВС, ЦЧС, ДЦУП, соответствующие технические службы

### 5.3 Уровень угрозы: Средний

**Характеристики:**
- Задержки движения
- Незначительные повреждения
- Нет угрозы жизни

**Требуемые действия:**
- Информирование соответствующих служб
- Планирование восстановительных работ
- Координация по необходимости

**Ключевые подразделения:**
- ДЦУП, соответствующие технические службы

### 5.4 Уровень угрозы: Низкий

**Характеристики:**
- Незначительные нарушения
- Минимальные повреждения
- Нет влияния на движение

**Требуемые действия:**
- Регистрация нарушения
- Планирование ремонта
- Выполнение в плановом порядке

**Ключевые подразделения:**
- Соответствующие технические службы

## 6. Документооборот и отчётность

### 6.1 Обязательные документы при аварии

| Документ | Ответственное | Срок |
|----------|---------------|------|
| Акт об аварии | ДАВС | Немедленно |
| План ликвидации | ДАВС | В течение 30 минут |
| Отчёт о ситуации | Оперативный штаб | Ежечасно |
| Акт о выполненных работ | Исполнитель | По завершении |
| Финансовый отчёт | Бухгалтерия | В течение 3 дней |
| Аналитический отчёт | Аналитики | В течение 7 дней |

## 7. Обучение и учения

### 7.1 Регулярные учения

| Тип учений | Периодичность | Участники | Цель |
|------------|---------------|-----------|------|
| Тактические учения | Ежемесячно | ДАВС, ДЦУП, технические службы | Отработка действий при типовых авариях |
| Комплексные учения | Ежеквартально | Все подразделения | Отработка взаимодействия при крупных авариях |
| Специальные учения | По необходимости | Специализированные подразделения | Отработка действий при специфических угрозах |

### 7.2 Требования к обучению

- Все сотрудники должны проходить обучение не реже 1 раза в год
- Диспетчеры — не реже 2 раз в год
- Руководители — не реже 2 раз в год
- Специалисты по ЧС — не реже 3 раз в год
